## Client

- Added `/analyse` route and sidebar item under the dashboard module.
- Added `AnalysePage` with shared period/basis controls and six tabs: Vue
  globale, Employes, Clients & Referents, Services, Creances & Engagements,
  and Rapports.
- Added `ChartCanvas`, a Chart.js wrapper that registers line/bar/doughnut
  controllers and destroys instances cleanly under React StrictMode.
- Added analysis tabs:
  - `GlobalTab`: summary, CA composition, and CA trend.
  - `EmployeesTab`: employee KPI chart/table with drill-down links.
  - `ClientsReferrersTab`: client and referrer KPI comparison charts/tables.
  - `ServicesTab`: service composition, volume/value framing, and service
    trend chart.
  - `CreancesEngagementsTab`: receivables by party, accrual-vs-cash
    comparison, and open invoice/proforma engagements.
  - `RapportsTab`: selectable report modules with Excel/PDF download links.
- Extended `reporting.api.ts` with CA/service trend calls, creance/open
  engagement analysis calls, and optional module lists on export URLs.
- Extended `creanceApi.list()` with an optional `partyId` filter.
- Extended party-history client types with commercial proforma/invoice rows.
- Fixed `AnalysePage.tsx`'s `RapportsTab` import casing after typecheck caught
  the mismatch on Windows.

## Server

- Expanded `/api/reporting` with:
  - `/ca-trend`
  - `/service-trend`
  - `/creances-by-party`
  - `/accrual-vs-cash`
  - `/open-engagements`
- `CaCompositionBucket` now includes `volume`, so "most used" services can be
  distinguished from highest value.
- Excel/PDF exports now accept module selection (`global`, `employes`,
  `clients_referents`, `services`, `creances`) and build matching sheets/PDF
  sections.
- Reporting PDF template is purpose-built for analysis exports instead of
  reusing the invoice print template shape.
- Party-history commercial section is now real: it lists invoices and
  proformas for the party, with proforma totals computed from `proformaLines`.
- Creance service/controller accepts `partyId` while preserving the single
  receivables aggregation query used by dashboard/reporting/party detail.
