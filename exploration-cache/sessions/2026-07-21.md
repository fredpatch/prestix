# Session 2026-07-21 — Sprint 11c-3 Close-out

## Summary

Closed Sprint 11c-3 (UI Hardening: Architectural Migration) in full — the
largest single-sprint push of the UI hardening initiative. Work landed in 16
incrementally-validated diff batches, each independently confirmed via
`git apply --check` + `npm install` + `npx tsc --noEmit` against a fresh
clone before handoff.

## What shipped

**RHF dialogs (carried over from a prior session, verified/corrected here):**
`CreatePartyDialog`, `EditPartyDialog` (shared `party-schema.ts`),
`CreateStockArticleDialog`, `RecordPaymentDialog`, `CreateCommissionDialog`.

**Generic table components:** `ReadOnlyTable` and `DataTable`, initially on
raw `<table>` markup, then rebuilt on shadcn's `Table` primitives once that
component was added mid-sprint. `ReadOnlyTable` grew a `ReactNode` title, an
optional `footer` slot, and a `bare` mode as real usage surfaced needs beyond
the original design.

**React Query hooks migration** — the bulk of the sprint:
- `hooks/queries/` + `hooks/mutations/`, one hook per query/mutation.
- Every page's data-fetching migrated: `PartyDetailPage`, `InvoiceDetailPage`,
  `ProformaDetailPage`, `CommissionEditQueuePage`, `CommissionsPage`,
  `CreancesPage`, `UsersPage`, `EmployeeActivityDetailPage`, `PartiesPage`,
  `InvoicesPage`, `ProformasPage`, `StockArticlesPage`, `DashboardPage`, and
  6 `analyse/*Tab.tsx` files.
- ~15 dialog mutations converted from plain async+manual-invalidate to real
  `useMutation`, activating the previously-dead global error toast.
- Every genuine list/detail table migrated to `DataTable`/`ReadOnlyTable`.

## Bugs found and fixed along the way

- **`UsersPage` OTP-reset and `CommissionsPage` delete had no error handling
  at all** — unhandled promise rejections on failure, silently swallowed.
  Surfaced by the `useMutation` migration; both now toast correctly.
- **Stale handoff claim corrected**: an earlier session's memory/handoff said
  "11 pages migrated to React Query." Actual count was 8, and 3 dialogs
  described as "migrated in sandbox" had never been committed — sandbox
  containers are ephemeral and don't persist between chat sessions. All
  redone and verified.
- **Process bug on my end, twice**: generated a diff from a stale local
  working copy that still had an already-merged change staged, causing a
  patch-apply conflict on Fred's machine. Root-caused and fixed by always
  pulling into a *fresh* clone immediately before generating any diff, never
  reusing a working copy across turns.
- **Self-caught regression before shipping**: an `IssueInvoiceDialog` hook
  draft would have offered the manager-only stock-override UI to any role,
  not just manager+. Caught during the caller review and fixed by threading
  `canOverrideStock` into the hook rather than deciding inside it.

## Deliberate scope boundaries (not oversights)

- `InvoiceDetailPage`/`ProformaDetailPage` line-items tables — inline
  row-editing state doesn't map cleanly onto `DataTable`.
- `CommissionEditQueuePage`'s per-request comparison grid — a 3-column
  current/proposed diff per card, not a record list.
- `SettingsPage` — sub-component-local loading pattern, skipped since 11c-1.
- `BootstrapPage` — one-time wizard, no cross-component cache-sharing need.

## Validation posture

manually smoke-tested in a running app this session — that's the natural
next validation step before calling 11c-3 fully done in practice, not just
